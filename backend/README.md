# AI Cost Detective Backend

FastAPI backend foundation for an AI-powered Azure FinOps platform. The service uses Azure SDKs only for Azure integration, PostgreSQL for persistence, SQLAlchemy 2.x for data access, and Alembic for migrations.

## Features

- Azure authentication with `DefaultAzureCredential` and service principal fallback.
- Azure subscription and resource group discovery.
- Resource discovery through Azure Resource Graph.
- Azure Cost Management integration for cost and usage retrieval.
- Azure Advisor integration for recommendation retrieval.
- Azure Monitor integration for utilization metrics.
- Deterministic FinOps rule engine for findings. AI is not used to generate findings.
- FinOps scoring engine.
- Redis Queue and background worker execution for long-running analysis.
- WebSocket progress updates for real-time dashboard feedback.
- Analysis creation workflow that queues work immediately and stores discovered resources, costs, findings, and scores from the worker.
- Health checks for database, Redis, and Azure authentication.
- Centralized JSON-friendly error responses and structured logging.
- Clean separation between API routes, services, repositories, schemas, and models.

## Local Setup

1. Create a virtual environment with Python 3.12.
2. Install dependencies:

```bash
pip install -r requirements.txt
```

3. Copy `.env.example` to `.env` and fill in values:

```bash
cp .env.example .env
```

4. Run database migrations:

```bash
alembic upgrade head
```

5. Start the API:

```bash
uvicorn app.main:app --reload
```

Swagger UI is available at `http://localhost:8000/docs`.

## Environment Variables

- `DATABASE_URL`: PostgreSQL connection string, for example `postgresql+psycopg://postgres:postgres@localhost:5432/ai_cost_detective`.
- `REDIS_URL`: Redis connection string, for example `redis://localhost:6379/0`.
- `AZURE_TENANT_ID`: Azure tenant ID for service principal fallback.
- `AZURE_CLIENT_ID`: Azure client ID for service principal fallback.
- `AZURE_CLIENT_SECRET`: Azure client secret for service principal fallback.
- `OPENAI_API_KEY`: Reserved for future AI analysis.

## API

- `GET /api/v1/subscriptions`: Lists accessible Azure subscriptions.
- `GET /api/v1/resource-groups/{subscription_id}`: Lists resource groups in a subscription.
- `GET /health`: Checks database, Redis, and Azure authentication.
- `GET /api/v1/analyses`: Lists all historical analyses for the current tenant.
- `POST /api/v1/analysis`: Creates an analysis record, queues background execution, and returns the analysis ID and job ID immediately.
- `GET /api/v1/analysis/{analysis_id}/status`: Returns status, progress percentage, current stage, and error details when applicable.
- `GET /api/v1/analysis/{analysis_id}/findings`: Lists generated FinOps findings for an analysis.
- `GET /api/v1/analysis/{analysis_id}/cost-summary`: Returns total monthly cost, potential savings, and resource count.
- `GET /api/v1/analysis/{analysis_id}/score`: Returns the calculated FinOps score.
- `WS /ws/analysis/{analysis_id}`: Streams progress events for an analysis.

## Background Worker

API requests do not execute long-running Azure analysis inline. `POST /api/v1/analysis` stores a queued analysis and submits an RQ job to Redis.

Run a worker process from the `backend` directory:

```bash
python -m app.workers.worker
```

The worker executes these stages:

- `resource-discovery` at 10 percent
- `cost-analysis` at 30 percent
- `advisor-analysis` at 50 percent
- `metrics-analysis` at 70 percent
- `finops-analysis` at 85 percent
- `scoring` at 95 percent
- `completed` at 100 percent

Failures are persisted on the analysis record and published over the analysis WebSocket channel.

## FinOps Rules

Findings are generated by deterministic logic only:

- Overprovisioned VM: CPU average below 10 percent and monthly cost greater than 50.
- Idle VM: CPU average below 2 percent across the 30-day metrics window.
- Unattached managed disk: disk has no compute attachment.
- Unused public IP: public IP has no associated IP configuration.
- Reserved instance opportunity: VM has stable utilization and monthly cost above the configured threshold.
- Advisor cost recommendation: Azure Advisor cost recommendations are converted into findings.

## Notes

All Azure operations are performed with Azure SDK clients. The codebase does not use OpenAI, LLM calls, Azure CLI commands, or mock cost data for finding generation.
